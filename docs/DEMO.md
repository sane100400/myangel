# 5분 시연 시나리오

심사위원 앞 시연 동선. 시간 한도 5분, 차별점 4가지를 모두 보여주는 것이 목표.

## 사전 준비

- 본인 계정에 잔액 200+ 크레딧 (Supabase에서 충전: `UPDATE user_credits SET balance = 300 WHERE user_id = '<your-uuid>'`)
- 시연용 베이스 이미지 1장 (인물 또는 풍경, 1024×1024 권장)
- 시연용 레퍼런스 이미지 1장 (스타일·소품)
- 브라우저 캐시 비우고 시크릿창 권장

## 동선

### 0. 인트로 (15초)
> "MyAngel은 한국어 사용자에 최적화된 AI 이미지 생성·편집 스튜디오입니다.
> Google과 OpenAI의 최신 이미지 모델을 한 화면에서 비교하고, 마커 기반으로 영역까지 정밀하게 편집할 수 있습니다."

랜딩 페이지 보여주며 4개 차별점 카드 가리킴.

### 1. 프롬프트 강화 (45초) — Grammarly 스타일

`/generate` 진입.
1. 프롬프트에 직접 입력: `예쁜 카페의 감성적인 분위기`
2. 우측의 **`프롬프트 강화`** 버튼 클릭
3. 약한 표현(`예쁜`, `감성적인`)에 밑줄이 그어짐
4. 아래 추천 카드에서 더 구체적인 대안 (예: "노을빛이 들어오는 통창의") 선택
5. **다듬기** 버튼으로 자연스러운 한국어로 재정리

> "AI가 약한 표현을 짚어주고, Grammarly처럼 클릭 한 번에 더 시각적인 단어로 교체합니다."

### 2. Multi-model 비교 (60초)

화질 버튼과 개수 슬라이더를 살짝 시연 (1K, 2장).

1. **모델: Nano Banana Pro** 선택 → "이미지 생성" 클릭
2. ~20초 대기 (대기 중 다음 멘트)
   > "Google의 Nano Banana Pro와 OpenAI의 GPT Image 2 둘 다 통합되어 있습니다.
   > GPT Image 2는 텍스트·로고·라벨 표현이 강점이고, 4K는 Nano Banana Pro 전용으로 분리했습니다."
3. 결과 2장 표시 → 자동으로 마이페이지 저장 배지 확인
4. 모델을 **GPT Image 2**로 전환 → 같은 프롬프트 재생성 (시간 단축을 위해 1장으로)
5. 두 모델 결과 비교

### 3. 마커 기반 정밀 편집 (90초) — 핵심 차별점

생성 결과의 **`편집`** 버튼 클릭 → `/edit` 자동 진입.

1. **`+ 교체`** 버튼으로 마커 1개 추가 → 사진 속 특정 영역(예: 의자)으로 드래그
2. 마커 에디터의 **업로드** 버튼 → 레퍼런스 이미지 업로드 → 현재 마커에 자동 바인딩
3. **편집 실행**
4. ~15초 후 결과 1장 표시 → 비교

> "자연어로 '의자를 빨간색으로'라고 하면 모델이 어떤 의자인지 매번 다르게 해석합니다.
> 저희는 영역을 그레이스케일 마스크에 op 코드로 인코딩해서 referent 분산을 0으로 만들었습니다.
> 같은 마커 = 같은 영역 = 같은 결과."

### 4. 글로벌 분위기·조명 조정 (30초)

마커 없이도 가능한 전역 조정 시연.

1. 마커 모두 삭제
2. 분위기 칩에서 **`드라마틱`**, 조명에서 **`황금시간`** 선택
3. **편집 실행**
4. 같은 사진이 톤·라이팅만 바뀐 결과

### 5. 크레딧 시스템 (30초)

상단 잔액 pill 가리킴.

> "모든 비용은 크레딧으로 투명하게 차감됩니다. Supabase RPC 단일 트랜잭션이라 동시 요청에도 race 없이 안전합니다.
> 가입하면 100크레딧 자동 지급."

`/boards` 마이페이지 보여주며:
> "결과는 자동으로 서버에 저장돼서 어떤 디바이스에서 로그인해도 같은 갤러리를 볼 수 있습니다."

### 6. 마무리 (15초)

랜딩으로 돌아가서:
> "Multi-model · 마커 정밀 편집 · 프롬프트 강화 · 크레딧 운영. 4가지가 한 자리에 모인 점이 핵심입니다.
> 감사합니다."

## 심사위원 예상 질문 + 답변

**Q1. "그냥 OpenAI/Google API wrapper 아니냐?"**
> "마커 기반 편집 프로토콜은 저희 고유 구현입니다. Nano Banana Pro 경로는 그레이스케일 마스크에 0/64/128/255로 op를 인코딩하고, GPT Image 2 경로는 알파 마스크를 전달합니다. 좌표 텍스트와 prompt builder로 같은 입력 계약을 일관되게 유지합니다. README의 docs/MARKER_PROTOCOL.md 참고 부탁드립니다."

**Q2. "보안은?"**
> "RLS + 명시적 user_id 필터 이중 방어, 매직 바이트 검증, SSRF 호스트 화이트리스트, CSRF Origin 검증, 단일 트랜잭션 크레딧 RPC. 위협 모델별 완화는 docs/SECURITY.md 정리, 단위 테스트로 검증."

**Q3. "비용 통제는?"**
> "운영자가 SQL 한 줄로 화질별·이벤트별 단가를 조절할 수 있고, 가입 보너스도 트리거에서 pricing_config를 읽어 즉시 반영됩니다. per-user rate limit으로 단일 사용자 abuse도 차단."

**Q4. "확장성은?"**
> "Next.js + Supabase 기반이라 수평 확장은 곧장 가능합니다. 현재 in-memory rate limit은 단일 노드 가정인데, Redis로 교체만 하면 멀티 인스턴스 환경 대응 가능합니다."

**Q5. "아직 안 된 게 뭐냐?"**
> "1) 부분 환불 — 4장 중 3장 성공 시 전액 차감, 정책 결정 단계. 2) 다각형 마커 — 1차는 원형만. 우선순위 따라 순차 적용."
